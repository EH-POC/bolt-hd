# Typography

Typography is for text writing including title, text and paragraph.

## Examples

### Typography.Title

There are 5 levels of a Title:

- **1** (for h1)
- **2** (for h2)
- **3** (for h3)
- **4** (for h4)
- **5** (for h5)

It is set to `1` by default.

#### Display text as your desired html tag

```jsx live
<>
  <Typography.Text tagName="label" fontWeight="bold">
    Products.
  </Typography.Text>
  <Typography.Text tagName="div">
    Manage HR, payroll, performance, learning, recognition and benefits easily
    with our integrated platform.
  </Typography.Text>
</>
```

#### Nested typography

#### Nested paragraph

When Typography.Text is renderned as paragraph block `p`, these following tags will be displayed followed hero-design setup:

| Typography.Text | Tag type | Supported nested tags                                                |
| --------------- | -------- | -------------------------------------------------------------------- |
| `p`             | inline   | `strong`, `b`, `label`: Rendered in semi-bold fontWeight by default. |
|                 | inline   | `a`: Rendered as primary Button.Link by default.                     |
|                 | inline   | `em`, `i`, `span`, `br`: Rendered followed default browser support.  |

<img src={TypographyP} alt="Nested paragraph" />

#### Nested division block

When Typography.Text is renderned as document division block `div`, these following tags will be displayed followed hero-design setup:

| Typography.Text | Tag type | Supported nested tags                                                |
| --------------- | -------- | -------------------------------------------------------------------- |
| `div`           | block    | `p`: Rendered as Typography.Text paragraph.                          |
|                 | block    | `ul`: Rendered with hero design customised padding-inline-start.     |
|                 | inline   | `strong`, `b`, `label`: Rendered in semi-bold fontWeight by default. |
|                 | inline   | `a`: Rendered as primary Button.Link by default.                     |
|                 | inline   | `em`, `i`, `span`, `br`: Rendered followed default browser support.  |

#### Important note

Only `first-level direct decendants` of the Typography.Text get effect, except:

- Typography.Text(as div) > p > inline tags (strong, b, a, em, i, span)

```JavaScript
// `a` tag is child of `Typography.Text` WILL get effect.
<Typography.Text>
  <a href="https://employmenthero.com/">
This will be formatted by hero-design.
  </a>
</Typography.Text>

// `a` tag is grand child of `Typography.Text` WON'T get effect.
<Typography.Text>
  <span>
    <a href="https://employmenthero.com/">
       This will NOT be formatted by hero-design.
    </a>
  </span>
</Typography.Text>

// Exception: `a` tag is grand child of `Typography.Text` div via a paragraph block.
// Since `p` is treated as `Typography.Text` inside the division block (div). This is the only exception of the rule.
<Typography.Text tagName="div">
  <p>
    <a href="https://employmenthero.com/">
  This will be formatted by hero-design.
    </a>
  </p>
</Typography.Text>
```
